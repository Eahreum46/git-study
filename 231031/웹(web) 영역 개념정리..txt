*웹(web) 영역 개념정리.

-현재 우리가 작업하는 모든 노력 및 결과물
-Web을 통해서 보여지는 결과물
-Web Browser 통하지 않으면 여러분들이 작업한 결과물은 절대 사용자에게 제공될 수 없음!
-웹브라우저= 클라이언트(request)
(ex.크롬, 엣지, 오페라, 사파리)
- 네이버= 포털사이트
-서버(respons)

서버는 클라이언트의 요청에 따라서 respons Message를 상황에따라 맞춰서 제공할수있다.
(*반드시 서버는 클라이언트에게 하나 이상의 메세지를 전달해야만 한다 => 웹 표준규약)


Message 종류

1) status 200: 클라이언트도 정상적으로 서버에 요청했고, 서버도 클라이언트의 요청에 정상적으로 피드백을 했다는 의미를 담고있는 메세지

2) status 303: 클라이언트의 정상적인 요청에 서버가 다른 대안을 갖고있는 웹 페이지로 리다이렉션을 진행한 경우

3) status 404: 클라이언트가 존재하지않는 도메인, 경로 URL을 서버측에 무리하게 요청한 경우

4) status 500: 클라이언트는 서버측에 정상적으로 데이터를 요청했으나, 서버에서 해당 데이터를 피드백할 수 없는 불가피한 상황
*서버가 다운,폭주,해킹
